# HH Auto Response (Chrome Extension)

Расширение для автоматического отклика на вакансии на hh.ru с учётом введённого сопроводительного письма и лимита количества откликов.

## Что делает
- На странице списка вакансий hh.ru перебирает карточки (`data-qa="vacancy-serp__vacancy"`).
- Кликает кнопку «Откликнуться» и при наличии текста добавляет сопроводительное письмо в модальном окне (`data-qa="vacancy-response-popup-form-letter-input"`).
- Жмёт «Откликнуться» в модалке (`data-qa="vacancy-response-submit-popup"`).
- После прохода по карточкам переходит на следующую страницу пагинации (`data-qa="pager-page"`) и продолжает до достижения лимита откликов.
  - Кнопку отправки ищет по `data-qa` и по тексту «Откликнуться», если разметка изменилась.
- Если кнопка отклика — ссылка, ведущая на отдельную страницу `/applicant/vacancy_response`, такая карточка пропускается. Если переход всё же случился, расширение вернётся назад и продолжит.

### Тайминг
- Между откликами ставится пауза 2–3 с + случайный джиттер 200–700 мс, чтобы имитировать ручные действия и снизить риск лимитов/капчи.
- При необходимости значения можно поправить в `content.js` в константе `TIMING`.

## Установка в Chrome
1. Откройте `chrome://extensions`.
2. Включите «Режим разработчика».
3. Нажмите «Загрузить распакованное» и выберите папку проекта `hh_response`.

## Использование
1. Авторизуйтесь на hh.ru, настройте фильтр и откройте страницу с результатами поиска вакансий.
2. Нажмите на иконку расширения:
   - Введите сопроводительное письмо (можно оставить пустым).
   - Задайте лимит откликов (по умолчанию 200).
   - Нажмите «Старт». Расширение будет работать на открытой вкладке, переходя по страницам.
3. Нажмите «Стоп» в попапе, чтобы прервать работу.

## Памятка
- Работайте на странице списка вакансий (результаты поиска), будучи залогиненным на hh.ru.
- Оставьте вкладку открытой: скрипт переходит по страницам пагинации и продолжает работу, пока не достигнет лимита или не остановите.
- Прогресс и статус видно в попапе; подробные логи — в консоли вкладки (`[HH Auto Response]`).
- Если hh начнёт замедлять ответы или просить капчу — увеличьте паузу в `content.js` (`TIMING`) или поставьте меньше лимит.
- Расширение не кликает по вакансиям, где не удаётся найти кнопку «Откликнуться».

## Замечания по реализации
- Состояние (лимит, прогресс, сопроводительное письмо) хранится в `sessionStorage` вкладки и восстанавливается после перехода на следующую страницу.
- Селекторы завязаны на `data-qa` атрибуты, а для кнопок «Откликнуться»/«Добавить сопроводительное» используется поиск по тексту на случай изменений классов.
- Лог работы пишется в консоль страницы (`[HH Auto Response]`), чтобы проще отлаживать.
